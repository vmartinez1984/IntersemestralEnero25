    Diagrama de un proyecto general

    						flowchart TD
						A[Frontend] --> B[Reglas de negocio]
						B --> C[Servicios]
						B --> D[Repositorios]
						B --> E[Almacenes]
    Donde
    El frontend, es la aplicación donde el usuario final interactua, ya sea una pagina web, applicación mobile.
    Las reglas de negocio, es donde se aplican validaciones, autorizaciones, comprobaciones, promociones etc.
    Repositorio es donde se persiste la data o tiene su origen, que puede ser un txt, excel, csv, json, base de datos sql, base de datos no sql.
    Servicio es un auxiliar a la reglas de negocios que ayuda a validar, verificar o procesar, por ejemplo en un proceso de enrolamiento o registro se sacan fotos a la ine y se manda aun servicio este devuelve la consistencia de los datos, algun parametro de autenticidad y los datos extraidos. Este es proporcionado por algun proveedor o tercero.
    Los almacenes es donde se guardan los archivos, por ejemplo las fotos del ine, documentación del cliente, contratos fimados por el cliente, imagenes, videos etc.
    En terminos ideales deben existir documentación de los procesos, manuales de los servicios, implementacion de los servicios, bases de datos, conexiones.
    Tip, en caso de no existir documentelos usted de la manera sencilla y rápida en la herramienta que tenga a su alcance, puede usar Notion, .md o alguna wiki.

    Analisis y planificación

    La fase de planificación incluye normalmente tareas como análisis de costos y beneficios, programación, estimación de recursos y asignación. El equipo de desarrollo recopila requisitos de varias partes interesadas, como clientes, expertos internos y externos, así como directivos, para crear un documento de especificaciones con los requisitos del software.

    El documento establece las especificaciones y define los objetivos comunes que ayudan a planificar el proyecto. El equipo estima los costos, define una programación y dispone de un plan detallado para conseguir los objetivos.

    En esta parte va lo documentación, aqui es donde el PM, se encarga de realizar los diagramas, las actividades, documentación interna de la empresa.

    Publicar en GithubPages

    Instalar git y crear una cuenta en Github.

    git init
    git add .
    git commit -m "Primer commit"
    git remote add origin https://github.com/tuusuario/tu-repositorio.git
    git push -u origin master

    Habilitar GitHub Pages

    Una vez que tu código está en GitHub:
    1. Ve a tu repositorio en GitHub.
    2. En el menú superior, entra en **Settings** (Configuración).
    3. Desplázate a la sección **Pages** (barra lateral izquierda).
    4. En la sección **Source**, selecciona la rama `main` (o `master`) y guarda.
    5. Si el código está en una carpeta específica (ej., `docs` o `dist`), selecciona esa carpeta como raíz.

    Esperar la publicación
